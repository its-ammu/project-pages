# AGENTS.md

## Cursor Cloud specific instructions

### Overview

This is a Next.js 16 task manager app ("projtweek") with Supabase for auth and PostgreSQL database. It is a single-service application (no monorepo).

### Services

| Service | How to run | Notes |
|---------|-----------|-------|
| Next.js dev server | `npm run dev` | Runs on port 3000 |
| Local Supabase | `npx supabase start` | Requires Docker; runs Postgres on 54322, API on 54321, Studio on 54323 |

### Key commands

- **Lint**: `npm run lint` (ESLint 9; has 2 pre-existing `react-hooks/set-state-in-effect` errors and 1 unused-var warning)
- **Build**: `npm run build`
- **Dev**: `npm run dev`
- **Supabase status**: `npx supabase status` (shows local credentials)
- **Supabase migrations**: Applied automatically on `npx supabase start`; files in `supabase/migrations/`

### Local Supabase setup

Docker must be running before `npx supabase start`. The local config (`supabase/config.toml`) has email confirmation disabled (`enable_confirmations = false`), so sign-up works instantly without confirmation emails. The Supabase anon key and URL for `.env.local` can be obtained from `npx supabase status -o env` (fields `API_URL` and `ANON_KEY`).

### Environment variables

The app requires a `.env.local` file with:
- `NEXT_PUBLIC_SUPABASE_URL` — local: `http://127.0.0.1:54321`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — obtain from `npx supabase status -o env`

### Gotchas

- The `supabase/config.toml` is generated by `npx supabase init` and is gitignored. If it's missing, run `npx supabase init` before `npx supabase start`.
- Docker in this cloud VM requires `sudo dockerd` to start and `sudo chmod 666 /var/run/docker.sock` for non-root access.
- Two optional Supabase services (`imgproxy`, `pooler`) may show as stopped — this is normal and does not affect the app.
